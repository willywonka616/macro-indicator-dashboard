## Two real bugs found and fixed this round

1. **`EUROSTAT_FRESH_DAYS` (150d) was too tight.** It accounted for
   Eurostat's real ~113-day publication lag but forgot Eurostat dates a
   quarterly observation to the START of the quarter, not the release
   date — the same convention FRED's own 220d quarterly threshold exists
   for. The first CI round of this pass demonstrated this live: native
   `gov_10q_ggdebt` correctly resolved current, healthy 2026-Q1 data, and
   the guard wrongly rejected it as 204 days stale. Fixed to 230d; the
   second CI round confirms the debt row now ships live.
2. **ECB RAS's first confirmed-real key resolves to implausible values.**
   The exact series key discovered live in the prior round
   (`M.N.4F.1C.S121.S121.FC.FI.RT1.RT.F41A.TM13.EUR.X1.N.N.ALL`) returns
   HTTP 200 with real observations — but every value is exactly 0.0, not
   plausible aggregate reserve assets. A new `_plausible_reserves()`
   guard in `scripts/ecb.py` now rejects this before it can ship as a
   live "0% of GDP" figure. Confirmed both before (bug demonstrated) and
   after (guard correctly rejects it) via two separate live CI runs.
